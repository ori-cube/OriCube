# OrigamiPost v2 ドキュメント

Three.js を使用して折り紙の折り動作を再現するコンポーネント。ユーザーが頂点をドラッグ&ドロップすると折り線を計算し、板を分割・回転して繰り返し折ることができる。折り手順の履歴（`OrigamiStep` の列）を唯一の状態源とし、現在の形状は常に初期状態からのリプレイで導出する、という設計が全体を貫く原則になっている。

パスの表記はすべて `src/components/v2/OrigamiPost/` からの相対パス。各 `utils/` にはテスト（`index.test.ts`）が併設されており、仕様の実例集として読める。

## 読む順序

- はじめて読む場合は **01 → 02** を先に読む。以降のドキュメントはこの2つを前提にしている
- ユーザー操作の流れを追いたい場合は **03 → 04 → 05** の順で、1回の折りが確定するまでを時系列に追える
- **06〜08** はそれぞれ独立して読める。**09〜10** は 04〜06 の通常の折りを前提にした発展形。**11** は全ステップ種別のリプレイを前提にした保存レイヤー

## 既知の制限（現在の仕様）

- 埋もれた板を他の板を貫通して折れてしまう判定はしない（[06 破れ防止](./06-tear-prevention.md) の展開図連結性データは、将来この判定を実装するための基礎になる）
- 折り角は 180 度固定
- 開いて畳むは基本ケース（1つのポケットを対称に開いて畳む）限定（[09](./09-squash-fold.md)）
- 花弁折りは対称な基本ケース（フラップ2枚＋相方2枚）限定（[10](./10-petal-fold.md)）
- 中割り折り（インサイドリバースフォールド）は未対応（鶴の首・尾を作る操作。通常の折りの組み合わせでは再現しづらいことが確認されている）
- 折りが進むとスナップポイントが密集し、意図しない吸着が起きやすい。奥のレイヤーに埋もれた頂点も表示・吸着対象になっている（[03](./03-drag-and-fold-line.md) の既知の課題参照）
- v2 閲覧の再生・投稿フロー（保存 UI・API 接続）は未実装（[11](./11-procedure-export.md) の「今後」参照）
