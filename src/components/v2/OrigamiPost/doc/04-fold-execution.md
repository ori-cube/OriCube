# 04 折りの実行ロジック

`applyFoldStep`（`utils/applyFoldStep/index.ts`）が「現在の板群 + 1つの `FoldStep` → 折り後の板群」を計算する純関数で、リプレイの中核。折りが成立しない場合は null を返し、呼び出し側は折り全体を不成立として扱う。

## applyFoldStep のパイプライン

```
1. findFoldCandidates   dragVertexに頂点を持つ板を視点側から並べ、先頭foldCount枚を対象に
2. separateBoard        各対象板を折り線で2つの片に分割（1枚でも失敗なら全体不成立）
3. selectMovingBoard    各対象板の dragVertex と同じ側の片を「動く片」に
4. isTearingFold        紙が破れる折りなら不成立（→ 06 破れ防止）
5. rotateBoard          動く片を折り線周りに180度回転（= XY平面上の鏡映）
6. レイヤー再割り当て    動く片を視点側の外側に、元の重なり順を逆転して積む
```

対象の選び方（手順1）と複数枚の扱いは [05](./05-multi-board-folding.md) を参照。ここでは幾何計算（2, 3, 5, 6）を説明する。なお**対象外の板は折り線が幾何的に横切っていても動かさない**。対象の紙だけをつまんで折る、というセマンティクスのため。

## 板の分割 — separateBoard

頂点列を一周しながら、各頂点を折り線のどちら側にあるかで2つの片へ振り分ける。

- 側の判定は `isPointLeftOfLine`: 直線方向ベクトルと点方向ベクトルの**外積の z 成分**の符号で左/右を決める3値判定（|外積| < 1e-6 なら「線上」で null）
- 線上の頂点は**両方の片**に追加する
- 辺が折り線を跨ぐ場合は交点を計算して**両方の片**に追加する。交点のパラメータ t は、辺の両端の折り線からの符号付き距離の比 `t = d_start / (d_start - d_end)` で求める（両端が反対側にあるので分母は 0 にならない）

元の頂点の並び順のまま振り分けるため、**分割後のソートが不要**で、凸多角形の仮定もいらない（凹多角形にも対応）。

`sourcePolygon` は同じインデックス・同じ補間比率 t で**ロックステップ分割**する。側の分類・交点計算は折り畳み空間（`polygon`）だけで行い、展開図側は結果だけを同期させる。これにより分割後も両ポリゴンの頂点対応（[02](./02-data-model.md)）が保たれる。

分割不可として null を返すケース: 折り線が板を横切らない（片方の片の頂点が3未満）、折り線が辺と一致する、面積がほぼ 0（< 1e-6）の退化した片ができる、折り線自体が退化している。

## 動く片の決定 — selectMovingBoard

折り線に対して**ドラッグした頂点（dragVertex）と同じ側にある片**が動く。判定は `isPointLeftOfLine` の結果を比較するだけ。片の側は「折り線上にない最初の頂点」を代表に決める（分割済みの片は折り線を跨がないため、1頂点で全体の側が決まる）。

折り線はドラッグ前後の点の垂直二等分線なので dragVertex が折り線上に乗ることは通常ないが、乗っていた場合は防御的に null を返す。

## 180度回転 — rotateBoard と z=0 正規化

動く片の全頂点を `applyAxisAngle` で折り線（軸: `foldLine.start` を通る `end - start` 方向）周りに π 回転する。XY 平面上の板の 180 度回転は**折り線を軸とした鏡映**に等しい。

回転後、全頂点の z を明示的に 0 に揃える。浮動小数の誤差で z が微小にずれると、次のリプレイ時に側の判定などが揺れるため。`sourcePolygon` は折りで変換を受けないのでそのまま引き継ぐ。

なお、この回転はデータ上の確定計算であり、画面のアニメーションとは独立している。アニメーションはピボット Group の回転で別途行う（[08](./08-rendering-animation.md)）。

## レイヤー再割り当て

動く片は視点側の**外側**に積む。表折り（viewFront=true）なら現在の最大レイヤーの上、裏折りなら最小レイヤーの下。

積む順は**元の重なり順を逆転**させる。紙を折り返すと、上にあった紙ほど折った後は下に来るため。

```
例: layer 0, 1 の2枚重ねを両方表折りした場合
  折る前:  layer 1 (上)     →  動く片は layer 2  (下)
           layer 0 (下)     →  動く片は layer 3  (上)
  固定片はそれぞれ元の layer を保つ
```

戻り値は確定状態 `boards` に加え、アニメーション入力用に回転**前**の `movingBoards`（元のレイヤー付き）と、動かない `staticBoards`（対象板の固定片 + 対象外の板）も返す。
